# Proposal

## Change

將 `scenario/S01` 從線性分段教材改為真正的樹狀互動情境。

## Problem

- 目前 `S01` 的升階路線與降階路線同時寫在同一個 stage 文字裡，學習者無法直觀看到自己選擇造成的後果。
- 錯誤選項只會顯示 feedback，不會真的把情境推向更高風險的分支，訓練張力不足。
- 現有 `ScenarioPlayer` 只支援固定 `T0 -> T1 -> T2 -> T3`，無法表達團隊支援、保全介入、肢體衝突或不良後果。

## Outcome

- `S01` 可依選項真實跳轉到不同節點與不同結局。
- 錯誤選擇會導致情勢升溫，嚴重時進入公開衝突、保全介入或不良後果結局。
- 保留既有線性 scenario 相容，避免一次重寫 `S02–S04`。
